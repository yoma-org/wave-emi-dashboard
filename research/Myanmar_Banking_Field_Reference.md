## 4. MSISDN Format (Myanmar Mobile Numbers)

| Format | Example | Used by |
|--------|---------|---------|
| Local | `09781234567` | Wave Money, all mobile |
| International | `+959781234567` | International dialing |
| Without + | `959781234567` | Some systems |

### Operator Prefixes
| Operator | Prefixes |
|----------|----------|
| MPT | 092x, 094x, 095x |
| Telenor | 097x |
| Ooredoo | 098x, 099x |
| Mytel | 096x |
| MecTel | 093x |

### Validation Rules
- Must start with `09` (after normalization)
- Total: 10-11 digits
- Our system normalizes: `+959` → `09`, `959` → `09`, strips spaces/dashes

## 5. NRC Number Format (National Registration Card)

**Format:** `[State]/[District]([Citizenship])[Register]`

### State Codes (1-14)
| Code | State/Region |
|------|-------------|
| 1 | Kachin |
| 2 | Kayah |
| 3 | Kayin |
| 4 | Chin |
| 5 | Sagaing |
| 6 | Tanintharyi |
| 7 | Bago |
| 8 | Magway |
| 9 | Mandalay |
| 10 | Mon |
| 11 | Rakhine |
| 12 | Yangon |
| 13 | Shan |
| 14 | Ayeyarwaddy |

### Citizenship Types
| Code | Meaning |
|------|---------|
| C | Citizen |
| AC | Associate Citizen |
| NC | Naturalized Citizen |
| V | National Verification |
| M | Monk |
| N | Nun |

### Validation Regex
```javascript
/^\d{1,2}\/[A-Z]{3,6}\([A-Z]{1,2}\)\d{6}$/
```

### Examples
- `12/YANGON(N)123456` — Yangon, Nun, register 123456
- `7/BAGO(C)654321` — Bago, Citizen, register 654321

## 6. Bank Account Number Formats

| Bank | SWIFT | Account Format | Length |
|------|-------|---------------|--------|
| Yoma Bank | YOMAMMMYXXX | Numeric (e.g., 00554356200002) | 14 digits |
| KBZ Bank | KBZMMM | Numeric | 10-16 digits |
| CB Bank | — | Numeric | 12-16 digits |
| AYA Bank | — | Numeric | 10-14 digits |
| Wave Money | — | MSISDN (phone number) | 10-11 digits |

### How to Distinguish Phone from Bank Account
```javascript
function isPhoneNumber(value) {
  const normalized = String(value).replace(/[\s-]/g, '');
  return /^(\+?959|09)\d{7,9}$/.test(normalized);
}
// If isPhoneNumber → validate as MSISDN
// If NOT → treat as bank account number (skip phone validation)
```

## 7. Utiba CSV Formats (Wave Money)

### SalaryToMA (Mobile Account — direct wallet transfer)
```
AMOUNT,SOURCE_WALLET,TARGET_MSISDN,DESCRIPTION
100000,1200000147,09781234567,Corporate Disbursement~Client Name~ACME~Batch~1~Unique ID~EMP-001
```
- No header row
- TARGET_MSISDN = phone number
- Description uses tilde `~` separator

### SalaryToOTC (Over-the-Counter — agent payout)
```
AMOUNT,SOURCE_WALLET,AGENT_ID,DESCRIPTION,MSISDN,EMAIL,0,0
1000000,1200000147,52680,Corporate Disbursement~...,9895273008,ops@wavemoney.com.mm,0,0
400000,1200000147,52680,Corporate Disbursement~...,9895273008,ops@wavemoney.com.mm,0,0
```
- No header row
- **2 rows per employee** (1,000,000 + 400,000 MMK split)
- AGENT_ID default: 52680

### Fund Movement Files (3 types)
```
SOURCE,TARGET, AMOUNT ,DESCRIPTION
```
- DMM → Disbursement Ledger
- Disbursement → Corporate
- Corporate → Fee Disbursement

### Group Mapping Files (2 types)
```
MSISDN,GROUP NAME
09781234567,52680
```
- MapAgent: assign employee to OTC group
- UnmapAgent: remove from OTC group

## 8. Myanmar Language (Burmese Script)

### Unicode Range
- Myanmar block: U+1000 to U+109F
- Myanmar Extended-A: U+AA60 to U+AA7F
- Myanmar Extended-B: U+A9E0 to U+A9FF

### OCR Challenges
- Handwritten Myanmar script is extremely difficult — no reliable OCR solution exists
- Typed Myanmar text: GPT-4o and Gemini handle reasonably well
- Mixed English + Myanmar: most models can separate but accuracy drops
- Connected script (like Arabic) — character segmentation is hard

### Practical Approach
1. Build with English first (proven to work)
2. Test typed Myanmar with enterprise GPT-4o/Gemini when accounts ready
3. Handwritten Myanmar = manual entry fallback (no AI handles this reliably)

## 9. Key Validation Rules Summary

| Field | Validation | Error message |
|-------|-----------|---------------|
| MSISDN | Starts with 09, 10-11 digits | "Must start with 09, expected 10-11 digits" |
| Bank Account | Numeric, 10-16 digits | "Invalid account number format" |
| NRC | Matches regex pattern | "Invalid NRC format (expected: STATE/DISTRICT(TYPE)NUMBER)" |
| Amount | Positive number, > 0 | "Amount must be greater than zero" |
| Currency | Must be "MMK" | "Only MMK currency supported" |
| Employee Name | Non-empty after cleaning | "Employee name is required" |
| TOTAL_RECORD | Must match actual row count | "Header says N employees but found M rows" |
